Storage


Apple stores TouchID information in something called an Storage Enclave

The Storage Enclave


This is included in every device's chip that contains TouchID. Apple describes this as an "advanced security architecture" that was deisgned to protect this type of information. Rather than storing the actual scans/images of your fingerprint, it stores a mathamatical representation of them, which Apple claims it "isn't possible for someone to reverse engineer your actual fingerprint image from this stored data." Once a match is identified, the Secure Enclave grants access almost immediately. This matching process occurs within milliseconds, balancing speed and security. Apple has optimized the TouchID algorithm to keep the False Acceptance Rate (FAR) exceptionally low, meaning that the chance of a wrong fingerprint being accepted is minimal, roughly 1 in 50,000 (Apple Inc., 2024).


The Secure Enclave is seperated from the main processor, and has it's own Secure Enclave Processor to provide computing power. Having this dedicated procesor helps "prevent side-channel attacks that depend on malicious software sharing the same execution core as the target software under attack."



When the device is started up, a random key is genrerated for the Memory Protection Engine to use. "Whenever the Secure Enclave writes to its dedicated memory region, the Memory Protection Engine encrypts the block of memory using AES in Mac XEX (xor-encrypt-xor) mode and calculates a Cipher-based Message Authentication Code (CMAC) authentication tag for the memory." (Apple Inc., 2024) When the Secure Enclave reads from memory, the Memory Protection Engine verfies the tag. If this verification fails, the Secure Enclave stops accepting requests.


The Secure Enclave can't be accessed by the OS on your device, or by any applications running on it. It is never stored on Apple servers, or backed up to iCloud or anywhere else, and it can't be used to match against other fingerprint databases. (Apple Inc., 2024)
